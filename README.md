# Song Board Client

音游直播点歌板 —— 纯前端 Web 应用，手机浏览器打开后叠加在摄像头画面上，配合屏幕抓取直播使用。

## 功能

- 摄像头画面作为背景，点歌板半透明叠加
- 通过弹幕指令点歌（QQ群/B站/抖音，需配合弹幕推送后端）
- 歌曲数据实时从 [落雪咖啡屋 API](https://maimai.lxns.net) 获取，支持别名搜索
- 点歌自动选择最高难度，支持难度等级筛选
- 设置页面直接在浏览器内操作，数据存储在 localStorage
- 可扩展的游戏适配器架构（当前支持舞萌 DX）

## 快速开始

```bash
npm install
npm run dev
```

开发服务器监听 `0.0.0.0`，手机可通过局域网 IP 访问。

页面底部有调试输入栏，可以直接输入指令测试，无需连接弹幕后端。

## 构建部署

```bash
npm run build
```

产物在 `dist/` 目录，可部署到 GitHub Pages 或任何静态托管服务。

## 弹幕指令

### 所有人可用

| 指令 | 说明 | 示例 |
|------|------|------|
| `点歌 <关键词>` | 搜索并点歌，默认最高难度 | `点歌 Oshama Scramble` |
| `点歌 <关键词> <难度>` | 指定难度点歌 | `点歌 Oshama Scramble EXPERT` |
| `点歌 #<ID>` | 按曲目 ID 精确点歌 | `点歌 #11451` |
| `取消点歌` | 取消自己最后一次点歌 | |
| `难度 <名称/+/->` | 修改自己最后点的歌的难度 | `难度 MASTER` / `难度 +` |
| `有没有 <关键词>` | 查询曲库中是否有某首歌 | `有没有 马叫` |

### 管理员指令

需要在设置页面配置管理员 ID。

| 指令 | 说明 | 示例 |
|------|------|------|
| `删除 <序号>` | 删除指定位置的歌 | `删除 3` |
| `删除 all` | 删除全部 | |
| `完成 <序号>` | 将指定歌曲标记为已完成 | `完成 1` |
| `清空` | 清空整个点歌队列 | |
| `暂停点歌` | 暂停接受点歌 | |
| `开放点歌` / `开启点歌` | 恢复接受点歌 | |
| `游玩 <序号>` | 标记某首歌为游玩中 | `游玩 1` |
| `游玩` | 取消游玩状态 | |

## 设置

在主页面点击右上角 ⚙ 进入设置页面：

- **连接** — 弹幕推送后端的 WebSocket 地址
- **摄像头** — 前置/后置切换
- **叠加层** — 透明度调节
- **点歌规则** — 游戏选择、队列上限、每人点歌上限、是否允许重复
- **难度筛选** — 限制只能点哪些难度等级的歌（不勾选则不限制）
- **管理员** — 配置拥有管理权限的用户 ID

## 弹幕推送后端

本项目是纯前端，不直接连接任何直播平台。需要配合一个弹幕推送后端使用，后端通过 WebSocket 向前端推送统一格式的消息：

```json
{
  "platform": "qq",
  "userId": "123456",
  "nickname": "用户名",
  "content": "点歌 Oshama Scramble",
  "timestamp": 1718150400000
}
```

后端需要自行实现，负责对接具体的弹幕源（NapCat/OneBot、B站直播、抖音等）。

## 技术栈

- Vue 3 + TypeScript + Vite
- 纯 CSS（无 UI 框架）
- IndexedDB 缓存歌曲数据
- localStorage 持久化设置和队列状态
